## ğŸ”§ Development

### Development Setup

```bash
# Clone repository
git clone https://github.com/Q-T0NLY/terminal-zsh.git
cd terminal-zsh

# Install development dependencies
./install.sh --dev

# Run tests
./scripts/test_all.sh

# Lint code
./unified_linter.zsh
```

### Project Structure Guidelines

- **src/core/** - Core system functionality, no UI dependencies
- **src/visuals/** - Visual effects and rendering
- **src/system_management/** - System operations and utilities
- **src/ui/** - User interface components
- **src/modules/** - Feature modules (independent)
- **docs/** - Documentation only
- **examples/** - Working examples for users

### Coding Standards

**ZSH Scripts:**
- Use `#!/usr/bin/env zsh` shebang
- Prevent multiple sourcing with guards
- Log important actions
- Include function documentation
- Use meaningful variable names
- Handle errors gracefully

**Python Scripts:**
- Follow PEP 8
- Type hints for functions
- Docstrings for classes and functions
- Use virtual environments

### Adding New Features

1. **Create feature module** in appropriate directory
2. **Register with orchestrator** in `nexus_hyper_meta_orchestrator.zsh`
3. **Add documentation** in feature file
4. **Update feature matrix** in `FEATURE_INTEGRATION_MATRIX.md`
5. **Add tests** if applicable
6. **Update this documentation**

### Testing

```bash
# Syntax check
zsh -n script.zsh

# ShellCheck (if available)
shellcheck script.zsh

# Test in isolated environment
zsh -i -c "source script.zsh && test_function"

# Integration tests
./scripts/integration_tests.sh
```

## ğŸ” Troubleshooting

### Common Issues

#### Installation Issues

**Problem: Permission denied**
```bash
# Solution: Make scripts executable
chmod +x *.sh
chmod +x UNIVERSAL_SETUP.sh install.sh
```

**Problem: Command not found**
```bash
# Solution: Source configuration
source ~/.zshrc
# Or
source /path/to/nexus_quick_reference.sh
```

**Problem: Missing dependencies**
```bash
# Solution: Install dependencies
./install.sh --deps
# Or manually install
brew install zsh git curl wget
```

#### Visual Issues

**Problem: Colors not displaying**
```bash
# Solution: Check terminal color support
echo $COLORTERM
tput colors

# Enable 256 colors
export TERM=xterm-256color
export COLORTERM=truecolor
```

**Problem: Animations laggy**
```bash
# Solution: Reduce FPS
export QNX_FPS=15

# Or disable animations
export QNX_ACTIVE=0
```

#### Performance Issues

**Problem: Slow startup**
```bash
# Solution: Optimize configuration
nexus_optimize_config ~/.zshrc

# Profile startup time
time zsh -i -c exit

# Disable heavy features
export NEXUS_VISUALS_ENABLED=0
```

**Problem: High memory usage**
```bash
# Solution: Clear caches
nexus_clear_caches all

# Reduce visual complexity
export QNX_NEURAL_LAYERS=3
export QNX_QUANTUM_STATES=8
```

#### Configuration Issues

**Problem: Config conflicts**
```bash
# Solution: Validate configuration
nexus_validate_config ~/.zshrc

# Test in isolated environment
nexus_test_config ~/.zshrc

# Restore from backup
nexus_restore_configs <backup_id>
```

**Problem: PATH not updating**
```bash
# Solution: Rebuild PATH
nexus_propagate_paths

# Or manually
export PATH="/usr/local/bin:/usr/bin:/bin:$PATH"
source ~/.zshrc
```

### Debug Mode

```bash
# Enable debug logging
export NEXUS_DEBUG=1
export NEXUS_LOG_LEVEL=DEBUG

# Run with verbose output
nexus_init --verbose

# View debug logs
tail -f ~/.nexus/logs/debug.log
```

## ğŸ“š API Reference

### Configuration Propagator API

#### nexus_propagate_paths()
Optimizes PATH with version manager detection.

**Usage:**
```bash
nexus_propagate_paths [detection_report]
```

**Features:**
- Detects and validates PATH entries
- Adds framework paths (Homebrew, etc.)
- Detects version managers (nvm, pyenv, rbenv, cargo, go)
- Removes duplicates and broken paths

#### nexus_propagate_aliases()
Generates smart aliases with conflict detection.

**Usage:**
```bash
nexus_propagate_aliases [detection_report]
```

**Alias Categories:**
- Safety aliases (rm -i, mv -i, cp -i)
- Productivity aliases (ll, la, ..)
- Git aliases (gs, ga, gc, gp, gl, gd)
- Docker aliases (dps, dimg, dex)
- Kubernetes aliases (k, kgp, kgs, kgd)

#### nexus_propagate_env_vars()
Manages environment variables.

**Usage:**
```bash
nexus_propagate_env_vars [detection_report]
```

**Variables Configured:**
- EDITOR/VISUAL
- PYTHONDONTWRITEBYTECODE/PYTHONUNBUFFERED
- NODE_PATH
- JAVA_HOME
- COLORTERM
- LESS
- TZ

#### nexus_propagate_completions()
Configures shell completion systems.

**Usage:**
```bash
nexus_propagate_completions [detection_report]
```

**Completion Systems:**
- ZSH completion system
- Git completions
- Docker completions
- kubectl completions
- npm completions
- fzf fuzzy completion

#### nexus_update_zshrc()
Safely updates .zshrc with validation.

**Usage:**
```bash
nexus_update_zshrc "additions"
```

**Features:**
- Automatic backup before modification
- Removes existing Nexus section
- Adds new configuration
- Validates syntax
- Restores on failure

#### nexus_backup_configs()
Creates comprehensive configuration backup.

**Usage:**
```bash
nexus_backup_configs
```

**Backed Up:**
- All shell configs (.zshrc, .zshenv, .zprofile, etc.)
- Tool-specific configs (.vimrc, .gitconfig)
- Timestamp-based versioning
- Compressed tar.gz format

#### nexus_restore_configs()
Restores configuration from backup.

**Usage:**
```bash
nexus_restore_configs <backup_id>
```

**Features:**
- Lists available backups
- Shows manifest preview
- Requires confirmation
- Creates backup before restore

#### nexus_validate_config()
Validates configuration files.

**Usage:**
```bash
nexus_validate_config <config_file>
```

**Checks:**
- Syntax validation (zsh -n)
- ShellCheck integration
- File size warnings
- Circular dependency detection

#### nexus_optimize_config()
Optimizes configuration for performance.

**Usage:**
```bash
nexus_optimize_config [config_file]
```

**Optimizations:**
- Removes duplicate PATH entries
- Removes excessive empty lines
- Removes duplicate aliases
- Suggests lazy loading
- Estimates startup time

#### nexus_learn_preferences()
Learns from command history.

**Usage:**
```bash
nexus_learn_preferences
```

**Analysis:**
- Command frequency analysis
- Top 10 commands identification
- Usage patterns
- JSON preference export

### Cleanup Engine API

#### nexus_cleanup_system()
Main cleanup orchestrator.

**Usage:**
```bash
nexus_cleanup_system <safe|deep|surgical> [categories]
```

**Modes:**
- safe: User confirmation, backup, trash
- deep: Aggressive cleanup, all caches
- surgical: Interactive file selection

#### nexus_safe_cleanup()
Safe cleanup with backup protection.

**Usage:**
```bash
nexus_safe_cleanup <categories>
```

**Categories:**
- TEMP: Temporary files
- CACHE: Package caches
- USER: User data
- APPLICATION: Application data

#### nexus_deep_cleanup()
Aggressive system cleanup.

**Usage:**
```bash
nexus_deep_cleanup <categories>
```

**Cleans:**
- All package manager caches
- Broken symlinks
- Old log files (30+ days)
- Docker system (if exists)

#### nexus_backup_critical()
Backs up critical system files.

**Usage:**
```bash
nexus_backup_critical [backup_id]
```

**Critical Files:**
- Shell configurations
- SSH config
- Git config
- .config directory
- .local/bin directory
